About Me

I'm Divya Jain, an AI/ML Engineer and Computer Science student at Jaypee University of Engineering and Technology, with real-world experience building machine learning systems that go beyond notebooks and actually ship.

  • Personal Finance App - Frontend Mentor Challenge

    Role:Frontend Engineer
    Live Project: personal-finance-natashasworld.vercel.app
    Code Repository: github.com/natashapl/personal-finance

    Personal Finance App is a comprehensive full-stack web application that helps users take control of their money by tracking transactions, managing budgets, and building savings goals. Built as a solution to a Frontend Mentor challenge, this project goes beyond the original requirements by implementing complete user authentication, a REST API backend, persistent cloud database storage, and a full end-to-end test suite.

    The application gives users a real-time financial dashboard with an overview of balances, budgets, and pots, alongside detailed transaction history with search, sort, and filter capabilities. A built-in demo account lets visitors explore all features with realistic pre-loaded data without having to create an account.

    My Contributions

    Full-Stack Application Development

    • Built a complete full-stack application from the ground up using Angular 21 for the frontend and Ruby on Rails 8.1 in API-only mode for the backend.
    • Designed and implemented the PostgreSQL database schema hosted on Supabase, with all data scoped to individual users for complete privacy isolation.
    • Deployed the frontend to Vercel and the Rails API to Render, configuring each service for production with proper environment variables and CORS policies.

    Authentication & User Management

    • Implemented JWT-based authentication with bcrypt password hashing, including login, signup, and a read-only demo account for visitors.
    • Built an Angular HTTP interceptor to automatically attach bearer tokens to all API requests without any component-level token handling.
    • Enforced demo account restrictions server-side with a require_non_demo! guard on all mutation endpoints.

    Modern Angular Architecture

    • Used Angular 21 standalone components exclusively - no NgModules anywhere in the codebase.
    • Managed all application state with Angular Signals (signal(), computed(), effect()), eliminating the need for RxJS-heavy state management.
    • Applied ChangeDetectionStrategy.OnPush throughout and removed Zone.js entirely in favour of zoneless change detection, improving Lighthouse desktop performance to 100 and mobile performance to 94.
    • Implemented lazy-loaded routes for all five feature pages to keep the initial bundle size minimal.

    Feature Pages & UI

    • Built five fully functional pages: Overview (with a custom SVG donut chart), Transactions (with pagination, search, sort, and category filter), Budgets (CRUD with per-category spend tracking), Pots (savings goals with add/withdraw money flows), and Recurring Bills (with paid and due-soon status indicators).
    • Implemented a collapsible navigation sidebar with a mobile bottom nav that adapts across breakpoints.
    • Built a global toast notification system using Angular Signals with auto-dismiss, success and error variants, and screen reader announcements via aria-live.

    Accessibility & Quality

    • Achieved Lighthouse scores of 100 for Accessibility, Best Practices, and SEO on both desktop and mobile.
    • Built a custom FocusTrapDirective that locks keyboard focus inside modals, moves focus to the first interactive element on open, and restores it to the trigger element on close.
    • Applied WCAG AA standards throughout: semantic HTML, ARIA roles, colour contrast, aria-busy on loading states, and role="progressbar" on savings bars.

    End-to-End Testing

    • Wrote a full Playwright E2E test suite covering all 25 test cases across auth flows, budgets CRUD, pots CRUD (including add and withdraw money), transactions CRUD, search and filter, and pagination - all passing.
    • Diagnosed and resolved several tricky Playwright failures including strict mode locator violations, race conditions between Angular's loading cycle and DOM assertions, and simultaneous toast interference with toContainText assertions.

    Personal Finance App demonstrates my ability to architect and deliver a complete full-stack application end-to-end - from database design and API development through to accessible UI, automated testing, and production deployment. The combination of Angular 21, Rails 8.1, Supabase, and modern tooling reflects current professional development practices.
